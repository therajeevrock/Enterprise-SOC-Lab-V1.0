Previous chapters explained Sysmon, Wazuh, and security logs. This chapter brings those concepts together by explaining how a SIEM platform collects, analyzes, and presents security events for monitoring and investigation.

---

# What is SIEM?

SIEM stands for **Security Information and Event Management**.

A SIEM platform collects security events from multiple systems, processes the collected data, and provides a centralized location for monitoring, searching, and investigating security activity.

Instead of reviewing logs on individual machines, security analysts use a SIEM platform to monitor their entire environment from a single interface.

---

# Why is SIEM Important?

Modern organizations generate thousands or even millions of security events every day.

Reviewing these events manually is impractical.

A SIEM platform helps by:

- Collecting logs from multiple sources.
- Organizing security events.
- Detecting suspicious activity.
- Generating alerts.
- Supporting incident investigations.

Within the Enterprise SOC Lab, **Wazuh** serves as the SIEM platform used to monitor and analyze Windows endpoint activity.

---

# Core Functions of a SIEM

Although different SIEM platforms provide different features, most SIEM solutions perform the following core functions.

| Function | Description |
|----------|-------------|
| Log Collection | Collects logs from endpoints, servers, and network devices. |
| Event Correlation | Combines related events to identify suspicious activity. |
| Alert Generation | Generates alerts when predefined detection rules are matched. |
| Event Search | Allows analysts to search historical security events. |
| Security Monitoring | Provides centralized visibility across monitored systems. |

These capabilities help security teams identify and respond to potential threats more efficiently.

---

# How Does a SIEM Work?

A SIEM continuously receives security events from multiple data sources.

The collected events are processed, indexed, and made available through a centralized dashboard where analysts can search, monitor, and investigate suspicious activity.

The overall workflow is straightforward.

```text
Endpoints
      │
      ▼
Log Collection
      │
      ▼
SIEM Platform
      │
      ▼
Event Processing
      │
      ▼
Alert Generation
      │
      ▼
SOC Analyst
```

# Benefits of Using a SIEM

Some of the key benefits of using a SIEM platform include:

- Centralized log management
- Faster threat detection
- Improved security visibility
- Simplified incident investigations
- Historical event analysis
- Better operational efficiency for SOC teams

---

# SIEM in a Security Operations Center (SOC)

A SIEM platform is one of the most important tools used by a Security Operations Center (SOC).

Instead of manually reviewing logs from individual systems, SOC analysts use a SIEM to monitor the entire environment from a centralized dashboard.

This allows analysts to detect suspicious activity, investigate alerts, and respond to security incidents more efficiently.

---

# How SOC Analysts Use a SIEM

A typical SOC workflow using a SIEM includes the following steps:

1. Collect security events from monitored systems.
2. Review alerts generated by the SIEM.
3. Investigate suspicious events.
4. Determine whether the activity is legitimate or malicious.
5. Escalate or respond to confirmed security incidents.

By following this workflow, analysts can quickly identify potential threats and reduce the time required to investigate security events.
