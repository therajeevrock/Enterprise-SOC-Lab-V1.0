# What is the Wazuh Agent?

The Wazuh Agent is installed on monitored endpoints to collect system events and securely forward them to the Wazuh Manager.

In this lab, it is deployed on the Windows 10 endpoint to enable centralized monitoring through the Wazuh platform.

---

# Why Install the Wazuh Agent?

Without the Wazuh Agent, the Windows endpoint cannot send security events to the Wazuh platform.

Installing the agent enables:

- Secure communication with the Wazuh Manager.
- Collection of Windows event logs.
- Endpoint monitoring.
- Future Sysmon event collection.
- Security alert generation inside the Wazuh Dashboard.

The Wazuh Agent acts as the bridge between the Windows endpoint and the Wazuh platform.

# Generate the Windows Agent Installation Command

The Enterprise SOC Lab uses the Windows installation command generated by the Wazuh Dashboard to download and install the Wazuh Agent.

This command includes the required configuration parameters, such as the Wazuh Manager IP address and the Windows agent name, allowing the endpoint to register with the Wazuh platform during installation.

## Generate the Installation Command

On the Wazuh Dashboard, navigate to:

```text
Wazuh Dashboard
    ↓
Endpoints
    ↓
Deploy New Agent
```

Configure the following values before generating the installation command.

| Configuration | Recommended Value |
|---------------|-------------------|
| Operating System | Windows |
| Wazuh Server Address | Ubuntu Server IP Address |
| Agent Name | Windows 10 Computer Name (e.g., `SOC-Win10`) |
| Group | Default |

After generating the command, copy it and run it in **Windows PowerShell** with administrator privileges.

## Run the Generated Command

Run the generated command in **Windows PowerShell** with administrator privileges.

```powershell
Invoke-WebRequest -Uri https://packages.wazuh.com/4.x/windows/wazuh-agent-4.14.6-1.msi -OutFile $env:tmp\wazuh-agent; msiexec.exe /i $env:tmp\wazuh-agent /q WAZUH_MANAGER='ubuntu-server-ip' WAZUH_AGENT_NAME='Agent-name'
```

The command automatically:

- Downloads the Wazuh Agent installer.
- Installs the Windows agent.
- Configures the Wazuh Manager address.
- Sets the Windows agent name.

# Common Deployment Issues

### 1. Agent does not appear in the Dashboard
* **Possible Cause:** Incorrect Wazuh Manager IP
* **Resolution:** Verify the Manager IP address configured during installation.

### 2. Agent status remains Disconnected
* **Possible Cause:** Network connectivity issue
* **Resolution:** Verify VMware network settings and confirm the Windows endpoint can reach the Ubuntu Server.

###  3. Wazuh Agent service is stopped
* **Possible Cause:** Service startup failure
* **Resolution:** Restart the Wazuh Agent service and verify its status.

###  4. Registration failed
* **Possible Cause:** Incorrect Manager configuration
* **Resolution:** Reconfigure the Manager address and reinstall the agent if necessary.

###  5. Windows Firewall blocks communication
* **Possible Cause:** Required ports are unavailable
* **Resolution:** Verify Windows Firewall rules and allow communication with the Wazuh Manager.
