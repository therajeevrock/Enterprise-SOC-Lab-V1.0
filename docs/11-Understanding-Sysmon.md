# What is Sysmon?

Sysmon (System Monitor) is a Windows system service developed by Microsoft as part of the Sysinternals Suite.

It continuously monitors system activity and records detailed events that help security analysts understand what is happening on a Windows system.

Unlike the default Windows logs, Sysmon provides richer telemetry, making it easier to detect suspicious or malicious activity.

---

# How Does Sysmon Work?

Sysmon runs continuously in the background after installation.

Whenever a monitored activity occurs, Sysmon records the event in the Windows Event Log.

The Wazuh Agent then collects these events and forwards them to the Wazuh platform, where they become available for monitoring, detection, and investigation.

```text
Windows Activity
        │
        ▼
Sysmon
        │
        ▼
Windows Event Log
        │
        ▼
Wazuh Agent
        │
        ▼
Wazuh Platform
```

---

# What Information Does Sysmon Collect?

Sysmon records a wide range of Windows activities that are useful for security monitoring and incident investigations.

Some of the most common activities include:

- Process creation
- Network connections
- File creation
- Registry modifications
- Driver loading
- DNS queries

This information provides detailed visibility into endpoint activity and helps security analysts identify suspicious behavior.

---

# Why Do SOC Analysts Use Sysmon?

Security analysts rely on Sysmon because it provides detailed endpoint telemetry that is not available in the default Windows event logs.

Some common use cases include:

- Monitoring process execution
- Investigating suspicious programs
- Detecting unusual network activity
- Tracking file and registry changes
- Supporting incident response and threat hunting

Sysmon helps analysts understand what happened on an endpoint and provides valuable context during security investigations.

# Where Are Sysmon Events Stored?

After installation, Sysmon records its events in the Windows Event Viewer.

The events can be viewed by navigating to:

```text
Event Viewer
    ↓
Applications and Services Logs
    ↓
Microsoft
    ↓
Windows
    ↓
Sysmon
    ↓
Operational
```

This log serves as the primary source of Sysmon telemetry before the events are collected by the Wazuh Agent.

---

# How Sysmon Supports Wazuh

Sysmon and Wazuh work together to provide detailed endpoint visibility.

The overall workflow is straightforward:

```text
Windows Activity
        │
        ▼
Sysmon
        │
        ▼
Windows Event Log
        │
        ▼
Wazuh Agent
        │
        ▼
Wazuh Manager
        │
        ▼
Wazuh Dashboard
```

Sysmon generates the events, while the Wazuh Agent collects and forwards them to the Wazuh platform for monitoring and analysis.
