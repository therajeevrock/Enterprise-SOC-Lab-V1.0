# What Are Logs?

A log is a record of an activity that occurs on a computer, application, network device, or operating system.

Every time a user logs in, starts a program, accesses a file, or performs another action, the system may generate a log entry.

Logs provide a historical record of system activity and help security analysts understand what happened, when it happened, and where it occurred.

---

# Why Are Logs Important?

Logs provide visibility into system activity and help organizations detect, investigate, and respond to security incidents.

Without logs, it would be extremely difficult to determine the cause of a security event or reconstruct an attack timeline.

For SOC analysts, logs are one of the primary sources of evidence during security investigations.

---

# Common Types of Security Logs

A Security Operations Center (SOC) collects logs from multiple sources to monitor system activity and detect potential security threats.

Some of the most common log sources include:

| Log Source | Description |
|------------|-------------|
| Windows Event Logs | Operating system events generated by Windows. |
| Sysmon Logs | Detailed endpoint telemetry generated by Sysmon. |
| Wazuh Alerts | Security events processed and generated by the Wazuh platform. |
| Application Logs | Events generated by installed applications. |
| Network Logs | Events generated by firewalls, routers, and other network devices. |

Each log source provides different information that helps security analysts understand what is happening within an environment.

---

# Windows Event Logs

Windows records a wide variety of system activities in the Event Viewer.

Examples include:

- User logon and logoff
- Service start and stop
- Application errors
- System events
- Security events

These logs provide the foundation for monitoring Windows systems.

---

# Sysmon Logs

Sysmon extends the default Windows logging by recording detailed endpoint activities.

Examples include:

- Process creation
- Network connections
- File creation
- Registry modifications
- DNS queries

These events provide richer telemetry for security monitoring and investigations.

---

# Wazuh Alerts

The Wazuh platform collects events from monitored endpoints and analyzes them using detection rules.

When suspicious or important activity is detected, Wazuh generates alerts that can be reviewed through the Wazuh Dashboard.

These alerts help SOC analysts identify and investigate potential security incidents more efficiently.

---

# How Logs Help SOC Analysts

Logs are one of the most valuable sources of information during a security investigation.

By reviewing logs, SOC analysts can answer important questions such as:

- What happened?
- When did it happen?
- Which user or system was involved?
- Which process was executed?
- Was the activity normal or suspicious?

The answers to these questions help analysts determine the scope and impact of a security event.

---

# Log Collection Workflow

Within the Enterprise SOC Lab, security events follow the workflow shown below.

```text
Windows Activity
        │
        ▼
Windows Event Log / Sysmon
        │
        ▼
Wazuh Agent
        │
        ▼
Wazuh Manager
        │
        ▼
OpenSearch Indexer
        │
        ▼
Wazuh Dashboard
```

This workflow enables security events generated on the Windows endpoint to be collected, processed, stored, and displayed for monitoring and investigation.
