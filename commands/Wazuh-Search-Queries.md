# Wazuh Search Queries

This document contains the Wazuh search queries used by SOC analysts during the Enterprise SOC Lab investigations.

The queries are organized by investigation and are intended to help analysts quickly locate, verify, and investigate Windows security events collected by Sysmon and Wazuh.

Each query has been tested in the lab and represents the common searches performed during a typical SOC L1 investigation.
---

## Investigation Whoami

The following Wazuh search queries are commonly used by a **SOC L1 Analyst** to investigate the execution of the Windows `whoami` command.

| Investigation | Search Query | Event ID | Purpose |
|---------------|--------------|----------|---------|
| Whoami (Primary) | `data.win.eventdata.image:*whoami.exe` | Sysmon Event ID 1 | Finds all executions of `whoami.exe`. |
| Whoami (Alternative) | `data.win.eventdata.commandLine:whoami` | Sysmon Event ID 1 | Finds events where the command line contains `whoami`. |
| Process Create Events | `data.win.system.eventID:1` | Sysmon Event ID 1 | Displays all Sysmon Process Create events for timeline analysis. |
| Same User Activity | `data.win.eventdata.user:<Username>` | Sysmon Event ID 1 | Checks what other processes were executed by the same user. |
| Same Endpoint Activity | `agent.name:<Agent_Name>` | Sysmon Event ID 1 | Reviews other events generated by the same endpoint. |
| Same Parent Process | `data.win.eventdata.parentProcessId:<Parent_Process_ID>` | Sysmon Event ID 1 | Finds other child processes created by the same parent process. |
