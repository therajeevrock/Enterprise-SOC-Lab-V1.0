# Dashboard 01 - SOC Overview

## Overview

The SOC Overview dashboard was created to provide SOC analysts with a centralized view of security events generated within the Enterprise SOC Lab.

Instead of navigating multiple Wazuh modules, analysts can quickly review alert volume, identify suspicious activity, and begin investigations from a single dashboard.

This dashboard serves as the primary monitoring interface for the lab and will be expanded throughout this project with additional widgets such as Critical Alerts, Alert Trends, MITRE ATT&CK mapping, and Top Detection Rules.

## Objective

- Build a centralized SOC monitoring dashboard.
- Display security alerts in real time.
- Reduce investigation time by providing quick visibility.
- Simulate an enterprise SOC monitoring environment.

# Widget 02 - Critical Alerts

## Objective

The **Critical Alerts** widget displays the total number of critical security alerts generated by the Wazuh Rule Engine during the selected time range.

This widget counts only alerts with **Rule Level 15 or higher**, helping SOC analysts quickly identify high-priority security incidents.

---

## Step 1 - Verify the Default Critical Alert Filter

Before creating the widget, verify the default rule level used by the Wazuh Overview dashboard.

Navigate to:

**Overview → Critical Severity**

Click the **Critical Severity** card to open the alert details.

> Verify that the data source is **wazuh-alerts-\*** (not **wazuh-archives-\***), since alert widgets should use the alerts index.

Also verify that the filter is configured as:

```
rule.level: 15 to Infinity
```

This confirms the correct severity range used by Wazuh for Critical Alerts.

## Step 9 - Add Alert Filters

The **Metric** visualization is now created, but it currently displays the total number of alerts from the **wazuh-alerts-*** index.

To display only **Critical** alerts, apply the required filters.

1. Click **Add filter**.
2. Add the following filters:

   - **Field:** `manager.name`
     - **Operator:** `is`
     - **Value:** `wazuh-server`

   - **Field:** `rule.level`
     - **Operator:** `is between`
     - **Value:** `15` to `Infinity`

After applying these filters, the metric will display only **Critical severity alerts**.

> **Note**
>
> The `manager.name` filter limits the results to alerts generated by the **Wazuh Manager**, while the `rule.level` filter ensures that only **Critical alerts (Level 15+)** are counted.

## Create High, Medium, and Low Alert Widgets

The process for creating the **High**, **Medium**, and **Low** alert widgets is identical to the **Critical** alert widget created in the previous section.

Since the same workflow is repeated, the step-by-step instructions and screenshots are not included again.

> **Note**
>
> In this lab, the visualizations were created directly from the **Enterprise SOC - Overview** dashboard using **Edit → Add → Create new → Visualization**.
>
> Although visualizations can also be created from the **Visualize** section first and then added to a dashboard, creating them directly from the dashboard keeps the workflow simpler and avoids switching between different pages.

Create the remaining widgets by following the same steps used for the **Critical** alert widget, changing only the alert severity filter, visualization title, and description.

| Severity | Rule Level | Visualization Title |
|----------|------------|---------------------|
| High | 12 – 14 | `SOC - High Alerts` |
| Medium | 7 – 11 | `SOC - Medium Alerts` |
| Low | 0 – 6 | `SOC - Low Alerts` |

The dashboard will now contain separate widgets for **Critical**, **High**, **Medium**, and **Low** alerts, providing a quick overview of alert severity distribution.

## Final Dashboard Overview

After creating the Critical, High, Medium, and Low alert widgets, the Enterprise SOC - Overview dashboard provides a consolidated view of alert severity levels.

This allows SOC analysts to quickly monitor the number of alerts across each severity category from a single dashboard.
