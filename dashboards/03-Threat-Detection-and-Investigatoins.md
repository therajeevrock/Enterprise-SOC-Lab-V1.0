# Dashboard 03 - Threat Detection & Investigations

## Overview

The **Enterprise SOC - Threat Detection & Investigations** dashboard provides SOC analysts with centralized visibility into security detections generated across the monitored environment.

Unlike the **Enterprise SOC - Overview** dashboard, which focuses on the overall security posture, and the **Windows Endpoint Security Monitoring** dashboard, which focuses on endpoint-specific detections, this dashboard is designed to analyze security alerts, identify detection trends, monitor alert severity, and support incident investigation through actionable security insights.

This dashboard serves as the primary workspace for analyzing security detections, recognizing abnormal activity, and prioritizing investigations within the Enterprise SOC Lab.

## Objective

- Monitor security detections generated by the Wazuh Rule Engine.
- Analyze alert severity and detection trends over time.
- Identify significant Windows security events that require investigation.
- Provide centralized visibility into detection data for faster incident analysis.
- Support SOC analysts during threat detection and security investigations.

---

## Dashboard Information

| Property | Value |
|----------|-------|
| Dashboard Name | Enterprise SOC - Threat Detection & Investigations |
| Platform | Wazuh Dashboard |
| Visualization Engine | OpenSearch Dashboards |

# Widget 01 - Detection Severity Distribution

## Objective

The **Detection Severity Distribution** widget displays the distribution of Wazuh alerts based on their rule severity levels.

This visualization helps SOC analysts quickly understand the overall severity of security detections and prioritize incident response efforts based on alert criticality.

---

## Why This Visualization?

Not every security alert requires the same level of attention. Some alerts represent low-risk events, while others indicate high-risk or critical security threats.

Visualizing the distribution of Wazuh rule levels enables SOC analysts to quickly assess the security posture of the environment, identify the proportion of high-severity alerts, and prioritize investigations accordingly.

> **Note:** Wazuh stores alert severity as numeric **rule.level** values. Higher values indicate more severe security detections.

---

# Widget 02 - Detection Trend Over Time

## Objective

The **Detection Trend Over Time** widget visualizes the number of Wazuh security detections generated over a selected time period.

This visualization helps SOC analysts identify spikes in security events, monitor detection trends, and recognize periods of increased suspicious activity across the monitored environment.

---

## Why This Visualization?

Security incidents rarely occur at a constant rate. A sudden increase in alerts may indicate malicious activity such as brute-force attacks, malware execution, persistence attempts, or attacker reconnaissance.

Monitoring detection trends over time enables SOC analysts to quickly identify unusual activity, correlate events with ongoing investigations, and determine when security incidents began or intensified.

# Widget 03 - Top Windows Event IDs

## Objective

The **Top Windows Event IDs** widget displays the most frequently occurring Windows Event IDs detected by the Wazuh Rule Engine.

This visualization helps SOC analysts quickly identify which Windows security events occur most frequently across monitored systems, enabling faster threat detection and more effective incident investigation.

---

## Why This Visualization?

Windows Event IDs represent specific operating system and security events generated by Windows and collected by Wazuh.

Monitoring the most frequent Event IDs enables SOC analysts to identify recurring security events, recognize abnormal activity, and understand which types of events are most active within the environment.

By highlighting the most common Event IDs, analysts can quickly prioritize investigations without manually reviewing large volumes of security alerts.
